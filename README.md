# FUTURE_ML_02
Support Ticket Classifier 🚀
This repository contains a self-contained Python program, task2.py, that uses machine learning to automatically sort and prioritize incoming customer support tickets. By reading the text of a complaint, the application instantly routes the ticket to the correct department and determines its urgency level, helping support teams eliminate manual bottlenecks and respond to critical issues faster. 
📌 What the Program DoesThe code in task2.py processes raw customer text and outputs two distinct predictions simultaneously:  Category Classification: Identifies the department the ticket belongs to (such as Account, Billing, Technical Issue, or General Query).  Priority Prediction: Estimates how urgently the issue needs to be addressed (High, Medium, or Low).  
⚙️ How it Works Inside task2.pyThe script executes a complete text-processing and machine learning pipeline end-to-end:  Built-in Sample Data: The file includes an embedded list of example customer support tickets—ranging from locked accounts to server crashes—allowing you to test the entire system instantly without downloading any external datasets.  Text Preprocessing: A custom cleaning function converts all text to lowercase, strips away disruptive punctuation, and removes common, uninformative English words (like the, is, and and). This leaves behind only impactful keywords like refund, locked, or down.  TF-IDF Vectorization: Because machine learning algorithms require numbers rather than sentences, the text is converted into numerical vectors. This mathematical technique weighs words based on how unique and important they are to a specific issue.  Dual Machine Learning Models: The file trains two independent Linear Support Vector Classifiers (LinearSVC)—one dedicated to learning category patterns and the other dedicated to learning priority triggers.  Evaluation Scorecard: The script automatically splits the data, trains the models, and prints a statistical report showing exactly how accurately the AI is performing.  Live Inference Test: To demonstrate real-time readiness, the program runs a brand-new, unseen sentence ("Hey! The server is completely down...") through the pipeline to prove that it can successfully detect an operational emergency instantly.  
🔮 Scalability & Next StepsWhile task2.py runs instantly using built-in examples, it is structured to scale seamlessly into a production environment:  Real Datasets: The sample data layout can be replaced with a simple command to load massive, real-world customer service datasets from a CSV file.  Live Automation: The real-time classification function is ready to be wrapped into a web API, allowing businesses to automatically triage live web forms, emails, or chat platform inputs
